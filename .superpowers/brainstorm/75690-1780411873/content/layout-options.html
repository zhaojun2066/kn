文档页面整体布局 — 你更喜欢哪种？

参考 ccswitch.io 的文档布局，结合 AI Profile Manager 的现有设计语言

A

左侧固定侧边栏 + 右侧内容

经典文档布局。左侧 260px 固定导航菜单（sticky），右侧主内容区独立滚动。适合内容层次深的文档，导航始终可见。

B

顶部 Tab 导航 + 二级侧边栏

顶部横向切换大分类（如 安装指南 / CLI 参考 / Desktop 应用），左侧显示当前分类的子菜单。适合分类少但每类内容多的情况。

C

纯单页滚动 + 目录锚点

所有内容在一个长页面上，右侧浮动迷你目录（TOC）高亮当前位置。类似 README 风格，简单直接但导航能力较弱。
