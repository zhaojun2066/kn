# 文档页面设计规格

> 日期: 2026-06-02 | 状态: 待审阅

## 1. 概述

为 AI Profile Manager 网站新增独立的文档页面 (`/docs` 路由)，提供结构化的产品文档浏览体验。参考 ccswitch.io 的侧边栏 + 内容区双栏布局，复用现有站点的 Precision Dark 设计系统。

### 设计目标

- 将 README.md 中的完整文档内容转化为结构化、可导航的文档站点
- 保持与主站（landing page）的品牌一致性
- 桌面端提供完整的导航体验（侧边栏 + TOC），移动端适配小屏
- 支持暗色/亮色主题切换

---

## 2. 页面布局

### 2.1 桌面端（>768px）

```

│  搜索框    │                                │           │

│  ▸入门指南 │                                │  · 概述    │
│   简介     │  [代码块]                       │  · 安装步骤 │
│   安装 ●   │                                │  · 验证    │

│   ...      │                                │           │
│            │  ─────────────────────────────  │           │
│  ▸Desktop  │  ← 简介    快速开始 →           │           │
│   ...      │                                │           │
│            │                                │           │
│  ▸场景示例 │                                │           │
│   ...      │                                │           │
│            │                                │           │

│   ...      │                                │           │

```

- **侧边栏**: 260px 固定宽度，sticky 定位，内容超出时内部滚动
- **内容区**: 最大宽度 760px，左右自动留白，底部有上下页导航
- **右侧 TOC**: 约 160px 宽度，sticky 定位，显示当前页 h2/h3 标题锚点，滚动高亮当前位置

### 2.2 移动端（≤768px）

- **侧边栏**: 变为抽屉式，通过 NavBar 上的汉堡菜单按钮触发，从左侧滑入覆盖
- **底部导航**: 固定底部栏，显示「上一页」/「下一页」按钮
- **TOC**: 隐藏（屏幕太窄）

## 3. 侧边栏菜单结构

### 3.1 菜单树（5 个分组 / 15 个页面）

```
🚀 入门指南
  ├── 简介
  ├── 安装
  └── 快速开始

📋 CLI 参考
  ├── 配置管理          (profile add/set/unset/remove/default)
  ├── 命令参考          (所有 profile 子命令一览表)
  ├── Shell Wrapper     (ai claude / ai codex 用法)
  └── 配置结构          (config.yaml 格式、env 字段说明)

🖥️ Desktop 应用
  ├── 安装与启动        (下载、macOS 签名问题解决)
  └── 功能与快捷键      (主要功能、快捷键一览表)

📖 场景示例
  ├── 多账号切换        (两个不同中转同时使用)
  ├── 不同项目不同 Key   (按项目区分 API key)
  └── OpenAI 兼容中转    (用第三方中转跑 Codex)

💡 更多
  ├── FAQ               (折叠式问答)
  ├── 故障排查           (安装问题、权限问题、签名问题)
  └── 卸载与重置
```

### 3.2 侧边栏交互

- **搜索框**: 顶部固定，输入关键词实时过滤页面列表
- **分组标题**: 点击可折叠/展开整组
- **当前页高亮**: 蓝色左边框 + 浅蓝背景 + 蓝色文字
- **悬停效果**: 文字颜色变亮
- **页面缩进**: 子页面比分组标题多 8px 缩进

## 4. 内容区设计

### 4.1 每个文档页面包含

1. **面包屑导航**: `分组名 / 页面标题`，分组名可点击回到分组首页
2. **页面标题**: `<h1>`，28px，粗体
3. **正文内容**: markdown 渲染，包含标题、段落、列表、代码块、提示框、表格等
4. **底部导航**: 分割线下方，左对齐「← 上一页」、右对齐「下一页 →」

### 4.2 内容组件

| 组件 | 说明 |
|------|------|
| 代码块 | 深色背景，顶部文件名/语言标签，右上角复制按钮，monospace 字体 |
| 终端窗口 | 三色圆点 + 标题栏 + 终端内容区，用于展示 shell 交互 |
| 提示框 (callout) | 左边框 + 浅色背景，info/warning/tip 三种类型 |
| 命令表格 | 表头灰色大写，隔行无分割，悬停高亮 |
| 步骤列表 | 有序列表，宽松行距 |

### 4.3 右侧 TOC

- 提取当前页面所有 `<h2>` 和 `<h3>` 标题
- 显示为缩进列表，h3 比 h2 多 12px 缩进
- 滚动时高亮当前可视区域的标题（IntersectionObserver）
- 点击标题锚点平滑滚动到对应位置
- 只在大屏显示（≥1280px）

## 5. 与主站的集成

### 5.1 路由

- 使用 `vue-router` 添加 `/docs` 和 `/docs/:page` 路由
- `/docs` 重定向到 `/docs/introduction`（简介页）
- 当前是纯前端 SPA，后续可迁移到 SSG

### 5.2 NavBar 改动

- 「文档」链接从 `#docs` 锚点改为 `/docs` 路由
- 当前在 `/docs` 路径下时，「文档」文字高亮

### 5.3 首页 DocsSection

- 保留现有的三 Tab 布局，但精简内容
- 每个 Tab 底部加 "查看完整文档 →" 链接指向对应 `/docs` 页面
- 保持视觉风格一致

## 6. 技术实现

### 6.1 技术栈

- **框架**: Vue 3 + TypeScript（与现有 site 一致）
- **路由**: vue-router 4（新增依赖）
- **构建**: Vite（与现有 site 一致）
- **样式**: 复用 `global.css` 中的所有设计 token

### 6.2 新增文件

```
site/src/
├── router/
│   └── index.ts              # 路由配置
├── views/
│   └── DocsPage.vue          # 文档页主视图
├── components/
│   ├── docs/
│   │   ├── DocsSidebar.vue   # 侧边栏（搜索 + 菜单树）
│   │   ├── DocsContent.vue   # 内容区（markdown 渲染）
│   │   ├── DocsTOC.vue       # 右侧本页目录
│   │   ├── DocsSearch.vue    # 搜索功能
│   │   ├── DocCodeBlock.vue  # 代码块组件
│   │   ├── DocCallout.vue    # 提示框组件
│   │   └── DocFooter.vue     # 文档页 footer
│   └── ... (现有组件保留)
├── data/
│   └── docs.ts               # 文档内容数据（静态 TS 对象）
└── types/
    └── docs.ts               # 文档相关类型定义
```

### 6.3 内容管理

初期方案：文档内容以 TypeScript 对象存储在 `data/docs.ts` 中，每个页面对应一个对象，包含：

```ts
interface DocPage {
  id: string
  group: string
  title: string
  content: string        // markdown 字符串
  prev?: string          // 上一页 id
  next?: string          // 下一页 id
}
```

后续可迁移到 `.md` 文件 + Vite `import.meta.glob` 动态加载。

### 6.4 颜色方案

严格复用 `global.css` 中定义的 CSS 变量，不新增颜色 token。关键颜色：

- 侧边栏背景: `var(--clr-bg-elevated)` 或 `#0d1117`
- 当前页高亮: `var(--clr-accent)` (#00c6ff)
- 代码块背景: `#0d1117`
- 内容区文字: `var(--clr-text)`
- 次要文字: `var(--clr-text-secondary)`

## 7. 状态与边界

### 7.1 加载状态

- 无异步加载（内容为构建时内联），页面切换为即时渲染
- 页面切换动画: 淡入 + 上移 8px（复用现有 `.reveal` 动画）

### 7.2 空状态

- 搜索无结果时显示 "未找到相关文档"
- 404 页面路径显示 "页面不存在" + 返回文档首页链接

### 7.3 移动端特殊处理

- 打开侧边栏抽屉时，背景加半透明遮罩，点击遮罩关闭
- 底部导航栏不遮挡内容区（内容区加 `padding-bottom`）
